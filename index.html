hhhhold!

Feel the insanity of user-generated content in your project by putting simple hhhhold! URLs in your code.

  • hhhhold.com/243x150/b

  • hhhhold.com/png/250

  • hhhhold.com/110x200/d/gif

Refresh examples

Combine as many as you need

Explicit input
  • /200

    A 200px square image

  • /500x200

    A 500px wide by 200px tall image

  • /wxyz

    The specific id of an image you enjoy

  • /jpg, /gif, /png

    Choose your image type, default is jpg

Non-explicit sizes
  • /s

    An image with width and height both randomly between 100px and 250px

  • /m

    An image with width and height both randomly between 250px and 500px

  • /l

    An image with width and height both randomly between 500px and 900px

  • /xl

    An image with width and height both randomly between 900px and 1280px

  • /r

    An image with width and height both randomly between 100px and 1280px

Modifiers
  • /b

    A bright image

  • /d

    A dark image

  • /w

    A wide image (for use with non-explicit sizes)

  • /t

    A tall image (for use with non-explicit sizes)


  • Source

    All images from FFFFOUND!. There might be the occasional Not Super Safe For Work image.

  • Attribution

    All images have an attribution URL that will take you to their FFFFOUND! source.

  • Multiple hhhhold!s

    The browser will only fetch the data for a URL once on page load. If you end up with a bunch of the same image on your site, you can append a random query to each (hhhhold.com/l/t?1, hhhhold.com/l/t?2, etc) to force the browser to fetch different images.

    Even better, check out the HHHHold.js repo for development needs. It solves this issue completely.
